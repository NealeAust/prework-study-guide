Prework Study Guide

HTML

  • Is used to create the structure of the webpage.
  • A developer should firstly think about and then prepare a sturdy structure.
  • Consists of elements that make up the type of content.
  • Each element performs a unique role.
  • Elements work together to build the structure.
  • The head element contain information about the webpage.
  • The body element represents visible content shown to the user.
  • Comprises of elements to perform straightforward and advanced functions - prepare list.
  • IMPORTANT - HTML has critical role in ensuring the webpage looks good and is functional.
  • Includes elements designed to assist in presentation i.e. different levels of headings, paragraphs, sections etc.
  • Elements can be nestled inside other elements, referred to as parent/child elements.
  • The closing tag includes a forward slash
  • HINT - Type very carefully and check what you have typed

CSS

  • Defines the colours, aesthetics, visual layout and organisation of the webpage.
  • Used to define style rules for specific elements using a wide range of options.
  • CSS class is an attribute to define a group of unique elements to apply unique formatting.
  • An element selector or tag selector selects all the elements by their specified name.
  • A class selector selects all elements with a specified class attribute.
  • Inline style uses a CSS rule to apply a unique style to an specific page element only.
  • Internal style sheet contains CSS rules for the particlular webpage in the head section of the HTML file.
  • External CSS style sheet is where the rules are in a separate file, linked using a link element.
  • Using a wild card symbol * will apply a rule to all elements on a web page.
  • A margin indicates how much space we want around the outside of the element.
  • A padding indicates how space we want around the content inside an element.
  • CSS style rules also apply to images and paragraphs.
  • Feedback from an end user who identified a bug is critical.
  • Debugging steps: identify what the code should do, locate error, fix bug, test and document process.
